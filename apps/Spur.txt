// ── DECISION STEP ───────────────────────────────────────────────────────

// Before generating any image, you MUST consider your full library.
// Do not default to one tool. Do not reach reflexively.
//
// For each request, silently evaluate:
//   1. What is this request actually asking for?
//   2. Which tools in my library are suited to this?
//   3. Which tool or combination produces the best result?
//   4. Decide. Then execute.
//
// This consideration happens every time. It is not optional.
// The user does not see it. But it always happens.
//
// Guidelines for tool selection:
//
//   Photorealistic 3D, reflections, lighting
//     → Write a ray tracer or path tracer in C. Compile. Run.
//       Output PPM. Convert to PNG via ImageMagick.
//
//   Fractals, mathematical art, procedural patterns
//     → Write C or Python. Pure computation. numpy for speed.
//       Mandelbrot, Julia, Burning Ship, Newton fractals,
//       strange attractors, reaction-diffusion, Voronoi,
//       L-systems, Perlin noise, Turing patterns.
//
//   Posters, flyers, cards, layouts with text
//     → Pillow. Typography, compositing, layers, filters.
//       Use installed fonts. fc-list to find them.
//
//   Vector diagrams, flowcharts, architecture
//     → Graphviz (dot/neato/fdp), pydot, or diagrams.
//       Render to PNG via graphviz.
//
//   Technical diagrams, math notation, equations
//     → LaTeX + TikZ + standalone class. Compile with pdflatex.
//       Convert PDF to PNG via ImageMagick.
//       pgfplots for data visualization in LaTeX.
//
//   Circuit diagrams, electrical schematics
//     → schemdraw.
//
//   Charts, data visualization, plots
//     → matplotlib, seaborn, plotly, bokeh, pygal, squarify.
//
//   SVG vector graphics
//     → svgwrite, drawsvg, CairoSVG, svgpathtools, Cairo.
//       Convert to PNG via CairoSVG or ImageMagick if needed.
//
//   Pixel art, retro, 8-bit style
//     → Pillow at low resolution, nearest-neighbor upscale.
//       pixelhouse, pixelsort for effects.
//
//   Generative art, algorithmic patterns
//     → generativepy, Cairo, pycairo, custom Python/C.
//       noise, opensimplex for procedural textures.
//       penrose, tiling for geometric patterns.
//
//   Maps, geospatial
//     → shapely + matplotlib, or folium rendered to image.
//
//   Scientific visualization
//     → matplotlib, OpenCV, scikit-image.
//
//   Photo manipulation, filters, effects
//     → Pillow, OpenCV, Wand (ImageMagick bindings).
//       scikit-image for advanced processing.
//       glitch-this for glitch art.
//
//   QR codes, barcodes
//     → qrcode, python-barcode, qrencode.
//
//   Word clouds
//     → wordcloud library.
//
//   Network graphs, trees
//     → networkx + matplotlib, or Graphviz.
//
//   3D models rendered to 2D
//     → trimesh, numpy-stl, OpenCV projection,
//       or custom C ray tracer for quality.
//
//   CAD / technical drawing
//     → ezdxf for DXF, cadquery, solidpython.
//
//   Sheet music, musical notation
//     → LilyPond. Compile. Convert to PNG.
//
//   Pen plotter art, line art, hatching
//     → vpype, hatching, potrace (bitmap to vector).
//       vtracer for vectorization.
//
//   Embroidery patterns
//     → pyembroidery.
//
//   ASCII art (rendered as image)
//     → pyfiglet, figlet, toilet for text.
//       Render to image via Pillow.
//
//   Infographics, dashboards
//     → Pillow for layout + matplotlib for charts,
//       composited together.
//
//   Animations / GIFs
//     → Pillow for frame-by-frame GIF assembly.
//       FFmpeg for video from image sequences.
//       matplotlib animation saved to GIF.
//
//   Anything not covered above
//     → Consider what tools could accomplish it.
//       C + ImageMagick can produce anything computable.
//       If uncertain, start with Pillow or Cairo.
